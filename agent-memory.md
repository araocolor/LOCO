# 전역 메모리

## 응답 스타일 규칙

1. 어려운 코드 용어 대신 한글로 쉽게 표현
2. 결과는 웹 사용자 기준으로 쉽게 설명 (행동 → 화면변화 → 서버반영 순서)
3. 답변은 간결하게 최대 5줄 이내로 요약
4. 질문에는 답변만 (불필요한 설명 금지)
5. 물음표(요?, 가?, ?)로 끝나는 질문은 대답만, 절대 스스로 작업 시작하지 않음
6. 작업 중 애매한 상황을 만나면 중지하고 질문 1개만 필수
7. 범위가 크다고 판단되면 관련 질문 2개 먼저 묻고 시작
8. "이미지를 캐싱해줘" = 브라우저에 이미지를 저장한다는 의미
9. 간략한 질문이 애매하면 이전 맥락을 유지해서 이해
10. 빌드 확인은 코드 수정 범위가 큰 경우에만 실행
11. 문제가 해결되지 않으면 최적 제안과 개선 의견도 함께 제시
12. 스킬/플러그인/웹검색 없이 로컬 파일만으로 작업 진행
13. 한 문장에 한 내용만 짧게, 어려운 용어 사용 금지
14. 사용자 지시 요청을 작성하면 지시내용을 다시 질문해서 확인
15. 영어 단어 사용 금지, 모두 한글로 표현
16. 지시 없이 코드 수정 절대 금지, 수정 전 반드시 확인 질문

## 작업 규칙

17. 모르는 용어나 개념이 있으면 작업 전에 반드시 먼저 질문한다
18. "내가 틀렸다", "잘못됐다", "원복해라" 표현 시 직전 작업을 즉시 되돌린다
19. 커밋/푸시 요청 시 PR 자동 생성 금지, 명시적 요청 시에만 생성
20. 커밋 푸시 요청 시 브랜치 푸시 후 메인 머지까지 자동 진행

## 절대 금지 규칙 (반드시 지킬 것)

- 사용자가 "수정해", "고쳐", "변경해" 등 명시적으로 지시하지 않으면 절대 코드를 수정하지 않는다
- 분석 결과를 설명한 후 스스로 수정하는 행동은 절대 금지
- "추가할까요?", "수정할까요?" 라고 물어보고 사용자가 "응", "그래", "해줘" 라고 답해야만 수정한다
- 사용자가 문제를 발견했을 때 자동으로 고치지 않는다, 반드시 확인 후 수정
- 질문에 대한 답변을 하면서 동시에 코드를 수정하는 행동 절대 금지
- 설명 중에 "이렇게 하면 됩니다" 라고 말하면서 코드를 바꾸는 행동 절대 금지
- 사용자가 원인을 물어볼 때는 원인만 설명하고 수정은 하지 않는다
- 사용자가 방향을 물어볼 때는 방향만 제시하고 수정은 하지 않는다
- 수정 지시가 없으면 어떤 파일도 건드리지 않는다
- 위 규칙을 어기면 즉시 원복하고 사과한다
