# 알림 시스템 구현 계획

## 개요

웹뷰 앱에서 채팅 알림과 알림페이지 알림을 구현한다.
웹 코드로 가능한 부분을 먼저 구현하고, 네이티브 푸시는 이후 별도 작업한다.

---

## 1단계: 채팅 알림 (웹 코드)

### 1-1. 실시간 새 메시지 감지
- Supabase 실시간 구독으로 새 메시지 수신 감지
- 현재 열려있는 채팅방이 아닌 다른 방의 메시지만 알림 대상
- 채팅방별 알림 끄기 설정 확인 (소리만 끔, 표시는 유지)

### 1-2. 알림음 재생
- 채팅 알림 전용 사운드 파일 준비 (카톡 스타일)
- 5초 내 중복 알림음 방지 (1회만 재생)
- 채팅방 알림 끄기 설정된 방은 소리 미재생
- 현재 해당 채팅방에 있으면 소리 미재생
- 다른 채팅방에 있을 때는 소리 재생

### 1-3. 하단 메뉴 뱃지 표시
- 메시지 메뉴 아이콘에 빨간점 표시
- 전체 안읽은 메시지 수 숫자 표시
- 채팅방별 안읽은 수 개별 표시

### 1-4. 채팅 탭 내부 뱃지
- 친구탭이 기본으로 열림 (현재 구현 확인 필요)
- 채팅목록 탭에 빨간점 + 안읽은 숫자 표시
- 채팅방 진입 시 해당 방 알림 해제

---

## 2단계: 알림페이지 알림 (웹 코드)

### 2-1. 실시간 새 알림 감지
- Supabase 실시간 구독으로 notifications 테이블 변경 감지
- 새 알림 도착 시 알림음(alert01) 재생
- 5초 내 중복 알림음 방지 (채팅 알림음과 공유)

### 2-2. 종 아이콘 뱃지 표시
- 알림 메뉴(종) 아이콘에 파란점 + 안읽은 숫자 표시
- 알림페이지 진입과 무관하게 유지

### 2-3. 알림 탭별 뱃지
- 각 탭(클래스/댓글/좋아요/기타)에 개별 파란점 + 숫자
- 해당 탭 진입 시 2초 후 파란점 해제
- 미진입 탭은 파란점 유지

### 2-4. 알림 항목 관리
- 알림 항목은 사용자가 직접 삭제 (자동 삭제 없음)
- 해당 탭에 머물고 있을 때 새 알림 도착 → 소리만 재생 + 탭 뱃지 표시 → 2초 후 해제

---

## 3단계: 네이티브 푸시 알림 (이후 작업)

### 3-1. Capacitor 푸시 플러그인 설치
- @capacitor/push-notifications 설치
- iOS 푸시 권한 요청 로직

### 3-2. 기기 토큰 관리
- Supabase에 device_tokens 테이블 생성
- 앱 시작 시 토큰 등록/갱신
- 로그아웃 시 토큰 삭제

### 3-3. 서버 발송
- APNs 인증서 설정 (Apple Developer 계정)
- Supabase Edge Function 또는 별도 서버에서 푸시 발송
- 새 메시지/알림 발생 시 해당 사용자 기기로 푸시 전송

### 3-4. 백그라운드 알림
- 앱이 꺼져있을 때 네이티브 알림 배너 + 소리
- 알림 탭하면 앱 열리면서 해당 화면으로 이동

---

## 4단계: 알림 설정 (이후 작업)

- 전체 알림 끄기/켜기
- 채팅 알림 끄기/켜기
- 알림페이지 알림 끄기/켜기
- 채팅방별 개별 알림 설정
- 알림음 종류 선택 (향후)

---

## 공통 규칙

| 규칙 | 내용 |
|------|------|
| 중복 방지 | 5초 내 알림음 1회만 재생 |
| 채팅 뱃지 색상 | 빨간점 + 숫자 |
| 알림 뱃지 색상 | 파란점 + 숫자 |
| 채팅 알림 해제 | 채팅방 진입 시 |
| 알림 표시 해제 | 해당 탭 진입 후 2초 |
| 알림 항목 삭제 | 사용자 수동 삭제 |

---

## 작업 순서

1. **1단계 먼저** → 채팅 알림 (웹 코드)
2. **2단계** → 알림페이지 알림 (웹 코드)
3. **3단계** → 네이티브 푸시 (Capacitor + APNs)
4. **4단계** → 알림 설정 페이지
