# 친구관계 로직 질의응답 정리 (2026-05-10)

## 문서 목적
- 이번 대화에서 합의된 친구/팔로잉/차단/블랙/숨김 로직을 한 번에 확인하기 위한 정리 문서
- 구현 전에 정책 기준을 고정하기 위한 기준안

## 1) 방향성(큰 틀)
- 폐쇄형 커뮤니티 기준으로 `친구신청 -> 승인` 흐름 채택
- 관계 축은 3개로 분리
  - 친구관계
  - 차단관계
  - 신고(블랙)관계
- 인스타 방식의 장점(단순한 관계 관리) 일부를 적용하되, 서비스 특성(감정/관계 맥락) 반영

## 2) 친구신청/친구상태 규칙
- 친구신청 대기 중에는 재신청 불가
- 신청취소는 가능
- 신청취소 후 재신청 가능 시점: `24시간(1일) 이후`
- 거절 기능은 사용하지 않음
- 양쪽 모두 신청이 성립되면 자동으로 친구완료 전환

## 3) 메시지 권한 규칙
- 메시지는 기본적으로 허용
- 단, `차단회원` 또는 `블랙회원` 관계에서는 제한
- 블랙의 경우: 신고당한 사람은 신고자에게 메시지 전송 불가, 신고자는 전송 가능

## 4) 블랙(신고) 규칙
- 블랙은 전역 제재가 아니라 `개인 기준 관계`
- 블랙신고 시(신고자 기준)
  - 해당 회원은 친구목록/팔로잉목록에서 제외
  - 대기중 탭으로 이동
- 블랙해제 시
  - 블랙 적용 직전 상태(팔로잉/일반회원 등)로 복귀

## 5) 차단 규칙
- 차단 상태는 개별 회원 단위로 적용
- 차단해제 시
  - 차단 적용 직전 상태(팔로잉/일반회원 등)로 복귀
- 상태 전환(예: 차단 <-> 블랙)은 직접 변경 가능

## 6) 숨김 규칙
- 숨김은 `친구 상태`에서만 적용 가능
- 숨김은 친구목록에서만 미노출
- 메시지/기타 기능에는 영향 없음
- 숨김해제 시 무조건 친구목록으로 복귀

## 7) 대기중 탭 규칙
- 대기중 탭은 `숨김 + 차단 + 블랙` 통합 목록
- 정렬 우선순위
  1. 숨김
  2. 차단
  3. 블랙
- 같은 그룹 내 정렬: 최신 처리순

## 8) 상태 중복 규칙
- 한 회원은 `숨김/차단/블랙` 중 `하나의 상태만` 가짐
- 중복 상태 동시 보관은 하지 않음
- 차단 또는 블랙 처리 시 해당 상태로 교체

## 9) 재설계 관련 결론(대화 기준)
- 팔로잉은 별도 상태값으로 두기보다 계산형으로 처리하는 방향 선호
- 관계/제재(차단/신고)를 분리해 충돌 최소화
- 해제 시 직전 상태 복원 규칙을 핵심 정책으로 유지

## 10) 추가 확정 사항 (추가 질문 3개 답변 반영)
- 상태 전환 허용: `차단 -> 블랙`, `블랙 -> 차단` 모두 가능
- 친구신청 취소 후 24시간 제한은 `취소한 당사자`에게만 적용
