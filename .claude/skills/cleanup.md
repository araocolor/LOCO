## 실행 조건
- 사용자가 `/cleanup` 명령어를 입력했을 때
- 사용자가 "수고하셨습니다"라고 말했을 때

## 작업 순서

1. `git diff HEAD` 로 현재 세션에서 수정된 파일 목록 확인
2. 수정된 파일에서 다음을 검토:
   - 사용하지 않는 변수, 함수, import 가 남아있는지
   - 삭제해야 할 코드가 남아있는지
   - 잘못 삭제되어 다른 기능에 영향을 주는지
3. 영향받는 파일도 함께 검토 (수정된 함수/컴포넌트를 사용하는 파일)
4. 발견된 문제를 목록으로 보여주기
5. 사용자 확인 후 수정 진행

## 규칙
- 수정 범위: 세션에서 변경된 코드와 영향받는 파일만
- 문제 발견 시 바로 수정하지 않고 목록을 먼저 보여준다
- 사용자가 승인한 항목만 수정한다
- 문제가 없으면 "정리할 내용 없습니다"로 끝낸다
